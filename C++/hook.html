hook的艺术，论hook的多种实现方式

2022-03-02- 2022-03-03
-
c++-c++

什么是hook

hook 是拦截目标函数调用的一种手段。通常是将原始方法调用重定向到我们指定的新方法，以便在原始方法执行前后可以执行某些操作，包括检查参数、打印日志、过滤某些调用等等。

vmt hook

替换虚函数表的内容，将vtable（函数指针数组）中某函数指针直接指向新的函数

detour hook

执行覆盖的字节，并在代码执行后跳回正确的位置。

trampoline hook

inline hook 的实现手段是直接修改目标函数的代码(即内联修改)，通常是用跳转指令替换目标函数代码的前几个字节。跳转指令让目标函数进行实际处理前被重定向。

inline hook 需要解决的问题：

  • hook 成功后，如何调用原先的旧函数
  • 无限递归问题
  • 多线程场景时的 race conditions
  • 指令修正

为什么需要trampoline

前面提到的两个问题，可以通过 trampoline 来解决

  • hook 成功后，如何调用原先的旧函数
  • 无限递归问题

假设有一个方法 MessageBoxA()，它可以显示一个对话框。假设我们 hook 了该方法，即，将其重定向到某个 Proxy。这个 Proxy 先打印出参数，再调用 MessageBoxA() 显示对话框。但是要注意，由于已经被 hook，所以正常调用 MessageBoxA() 时它又会重定向到 Proxy。这是一个无限递归调用，最后会导致程序因 stack overflow 而中止。

有一个直观的解决方案：在 Proxy 中先 unhook MessageBoxA() (即取消重定向)，然后调用 MessageBoxA()，然后再一次 hook MessageBoxA() (即打开重定向)。

上图中提供了另外一个更通用的解决方案：使用 trampoline。先将 MesasgeBoxA() 的头5个字节保存在 trampoline 中，再向 MessageBoxA() 的头5个字节写入 Proxy 的地址。这样一来，hook MessageBoxA() 之后，如果你想正常调用原来的 MessageBoxA()，你可以借助 trampoline 来完成。

c++代码:

//src被hook函数的地址，dst为hook的函数地址
bool Hook(char* src, char* dst, int len) {
	if (len < 5) return false;
	DWORD oProc;
	VirtualProtect(src, len, PAGE_EXECUTE_READWRITE, &oProc);
	//nop掉原来的，只剩下jmp到hook函数
	memset(src, 0x90, len);
	uintptr_t relAddy = (uintptr_t)(dst - src - 5);
	//jmp
	*src = (char)0xE9;
	*(uintptr_t*)(src + 1) = (uintptr_t)relAddy;
	VirtualProtect(src, len, oProc, &oProc);
	return true;
}

//src被hook函数的地址，dst为hook的函数地址
char* TrampHook(char* src, char* dst, unsigned int len) {
	//构造gateway
	if (len < 5) return 0;
	//申请新内存，用于存放trampoline函数内容，多五个字节存放jmp命令
	char* gateway = (char*)VirtualAlloc(0, len + 5, MEM_COMMIT | MEM_RESERVE, PAGE_EXECUTE_READWRITE);
	memcpy(gateway, src, len);
	//jmp距离为目的地址-（当前函数地址+5），跳转到原函数
	//(src+len)-(gateway+len+5) ==src - gateway -5
	uintptr_t jumpAddy = (uintptr_t)(src - gateway -5);
	//trampoline内容之后指令为jmp 原函数
	*(gateway + len) = (char)0xE9;
	*(uintptr_t*)(gateway + len + 1) = jumpAddy;

	if (Hook(src, dst, len)) {
		//返回trampoline函数地址
		return gateway;
	}
	else return nullptr;
}

检测方法

if(*function_A == 0xe9) {
    printf("Hook detected in function A.\n");
}

unsigned char *addr = function_A;
while (*addr == 0x90) addr++;
if(*addr == 0xe9) {
    printf("Hook detected in function A.\n");
}

push函数地址

使用push指令在栈上推送函数地址，retn指令弹出到栈顶指针中（从栈顶地址开始执行），此方法采用绝对地址

检测技术

unsigned char *addr = function_A;
if(*addr == 0x68 && addr[5] == 0xc3) {
    printf("Hook detected in function A.\n");
}

浮点数

类似于push/retn方法，我们在堆栈上推送一个虚拟值，然后用真实地址覆盖掉他，然后返回

这种技术的有趣之处在于以下几点;我们不是将要跳转到的地址存储为 32 位地址，而是将其存储为 64 位浮点数。然后我们读取它（使用fld指令），并将其转换为32位值（使用fistp指令）。

从32位地址构造64位浮点数非常容易

double floating_point_value = (double) function_B;

函数 B 的位置是我们的hook函数。调用原始函数是通过trampoline完成的，这种情况下，trampoline必须包含11个字节的命令

有许多浮点指令可能对这种情况有用。例如，可以通过将一个值乘以π（Pi，可以使用fldpi指令获得）来计算钩子地址。

MMX/SSE

此技术类似于使用浮点进行挂钩的技术。但是，我们在这里使用 MMX/SSE x86 扩展，而不是使用浮点。

与浮点技术一样，这两种技术都使用 push/retn 方法。第一种方法涉及 MMX 指令，尤其是movd指令。它允许，就像fistp指令（浮点指令）一样，从内存中读取值，并将值存储到堆栈中。第二种方法使用 SSE 指令，使用相同的movd指令。这两种方法之间的唯一区别是MMX指令在64位寄存器上运行，而SSE指令在128位寄存器上运行。（尽管这在我们的例子中并不有趣，因为movd指令允许读取和写入32位值。

无论哪种方式，由于此技术与浮点技术完全相同

间接跳转

跳转指令后跟随指向hook函数的指针，间接跳转更像Push、Retn方法

间接跳转占6字节

hook_func_ptr表示可以找到hook函数（例如函数 B）地址的地址。

调用指令

通过call指令调用hook函数，需要esp+4来维护栈平衡
